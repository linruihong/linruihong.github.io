hello world

Docker 允许你在容器内运行应用程序， 使用 docker run 命令来在容器内运行一个应用程序。

例子：输出Hello world

docker run ubuntu /bin/echo "Hello world"

结果：

$ docker run ubuntu /bin/echo "Hello world"
Hello world

各个参数解析：

  • docker：Docker 的二进制执行文件。
  • run：与前面的 docker 组合来运行一个容器。
  • ubuntu：指定要运行的镜像，Docker首先从本地主机上查找镜像是否存在，如果不存在，Docker 就会从镜像仓库 Docker Hub 下载公共镜像。
  • /bin/echo “Hello world”：在启动的容器里执行的命令

以上命令完整的意思可以解释为：Docker 以 ubuntu 镜像创建一个新容器，然后在容器里执行 bin/echo “Hello world”，然后输出结果。

“镜像”与“容器”的关系，类似于“类”与“对象”的关系

启动容器（后台模式）

使用以下命令创建一个以进程方式运行的容器：

docker run -d ubuntu /bin/sh -c "while true; do echo hello world; sleep 1; done"

输出结果：

$ docker run -d ubuntu /bin/sh -c "while true; do echo hello world; sleep 1; done"
bffeb0f011214f2416eeaa2f30ce0353adf1f59908fa7b63b05f41ee86d66237

在输出中，我们没有看到期望的”hello world”，而是一串长字符

bffeb0f011214f2416eeaa2f30ce0353adf1f59908fa7b63b05f41ee86d66237

这个长字符串叫做容器ID，对每个容器来说都是唯一的，我们可以通过容器ID来查看对应的容器发生了什么。首先，我们需要确认容器有在运行，可以通过 docker ps 来查看

docker ps

输出如下：

$ docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED              STATUS              PORTS               NAMES
bffeb0f01121        ubuntu              "/bin/sh -c 'while..."   About a minute ago   Up About a minute                       unruffled_pasteur
  • CONTAINER ID：容器ID
  • NAMES：自动分配的容器名称

在容器内使用docker logs命令，查看容器内的标准输出：

docker logs bffeb0f01121

输出结果：

$ docker logs bffeb0f01121
hello world
hello world
（。。。这里省去一千个hello world）

其中 bffeb0f01121 是容器ID位于开头的一部分，可以不需要使用完整的容器ID，便可以让 docker 命令找到该容器。

查看容器内的标准输出时，可以不使用容器ID，使用 docker ps 输出的容器名字（在 NAMES 那一列），也可以查看容器的输出：

docker logs unruffled_pasteur

输出结果：

$ docker logs unruffled_pasteur
hello world
hello world
（。。。这里省去一万个hello world）

停止容器

我们使用 docker stop 命令来停止容器：

docker stop bffeb0f01121

或者：

docker stop unruffled_pasteur

输出结果：

$ docker stop unruffled_pasteur
unruffled_pasteur

停止容器后，再使用 docker ps 查看运行中的容器，输出：

$ docker ps
CONTAINER ID        IMAGE               COMMAND             CREATED             STATUS              PORTS               NAMES

列出镜像列表

我们可以使用docker images来列出本地主机上的镜像

$ docker images
REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
ubuntu              latest              d355ed3537e9        3 weeks ago         119MB
training/webapp     latest              6fae60ef3446        2 years ago         349MB

各个选项说明:

  • REPOSITORY：表示镜像的仓库源
  • TAG：镜像的标签
  • IMAGE ID：镜像ID
  • CREATED：镜像创建时间
  • SIZE：镜像大小

同一仓库源可以有多个 TAG，代表这个仓库源的不同个版本，如ubuntu仓库源里，有15.10、14.04等多个不同的版本，我们使用 REPOSTITORY:TAG 来定义不同的镜像。

所以，我们如果要使用版本为15.10的ubuntu系统镜像来运行容器时，命令如下：

docker run -t -i ubuntu:15.10 /bin/bash

如果要使用版本为14.04的ubuntu系统镜像来运行容器时，命令如下：

docker run -t -i ubuntu:14.04 /bin/bash

如果你不指定一个镜像的版本标签，例如你只使用 ubuntu，docker 将默认使用 ubuntu:latest 镜像。

获取一个新的镜像

当我们在本地主机上使用一个不存在的镜像时 Docker 就会自动下载这个镜像。如果我们想预先下载这个镜像，我们可以使用 docker pull 命令来下载它。

例如，用 docker search 查询到了一个名为 mvertes/alpine-mongo 的 mongo 镜像，可以用如下命令下载：

docker pull mvertes/alpine-mongo

结果显示：

创建镜像

当我们从docker镜像仓库中下载的镜像不能满足我们的需求时，我们可以通过以下两种方式对镜像进行更改。

  • 从已经创建的容器中更新镜像，并且提交这个镜像
  • 使用 Dockerfile 指令来创建一个新的镜像

更新镜像


更新镜像之前，我们需要使用镜像来创建一个容器。

$ docker run -t -i ubuntu /bin/bash
root@277d67d262f7:/#

注意

使用 Docker Quickstart Terminal 启动终端输入窗口，使用上面的命令都是可行的。但是，直接使用 git 自带的 git-bash.exe 运行 docker 目录下的 start.sh 来启动时，输入上面的命令就会出错：

$ docker run -t -i ubuntu /bin/bash
the input device is not a TTY.  If you are using mintty, try prefixing the command with 'winpty'

改变命令，则是另一种错误：

$ winpty docker run -t -i ubuntu /bin/bash
docker: Error response from daemon: oci runtime error: container_linux.go:262: starting container process caused "exec: \"D:/Program Files/Git/usr/bin/bash\": stat D:/Program Files/Git/usr/bin/bash: no such file or directory".

网上说有两种可能：

  • The file or directory you are mapping is invalid
  • The base image you are using do not have /bin/bash

再修改命令，居然可以了，而且查了一下，容器里面的 /bin 下也有 bash 啊，不知道为什么上一个命令不能用。

$ winpty docker run -t -i ubuntu
root@b836e7b03121:/#

2017/7/24 补充：

从这里看到一个类似指令docker run -it ubuntu bash，也是可以的，与前面的指令结果一致，不过同样要在指令前面加winpty才行。

$ winpty docker run -it ubuntu bash
root@0cb8072550ec:/#

2017/7/27 补充：

在命令窗口中，使用 Docker Machine ssh 到虚拟机上，可以使用在虚拟上使用的命令，包括上面的docker run -t -i ubuntu /bin/bash（不需要使用变化过的指令，例如上面提到的winpty docker run -t -i ubuntu、winpty docker run -it ubuntu bash或者docker run -it ubuntu bash）。

在 Docker Quickstart Terminal 命令行窗口中，或者打开 Docker Toolbox 安装目录下的 start.sh ，在出现的命令行窗口中，输入命令docker-machine.exe ls可以查看当前管理的 Docker 主机：

在完成操作之后，输入 exit命令来退出这个容器。此时ID为 277d67d262f7 的容器，是按我们的需求更改的容器。

我们可以通过命令 docker commit来提交容器副本：

$ docker commit -m="update ubt" -a="homerlim" 277d67d262f7 homerlim/ubuntu:v1
sha256:5a32b6d303dde821d126c6bad8b6e6253f747f7171b10560a60e463bf008721b

各个参数说明：

  • -m：提交的描述信息
  • -a：指定镜像作者
  • 277d67d262f7：容器ID
  • homerlim/ubuntu:v1：指定要创建的目标镜像名

我们可以使用 docker images 命令来查看我们的新镜像 homerlim/ubuntu:v1：

$ docker images
REPOSITORY             TAG                 IMAGE ID            CREATED              SIZE

mvertes/alpine-mongo   latest              bcac9fb36a28        2 months ago         106MB
training/webapp        latest              6fae60ef3446        2 years ago          349MB

构建镜像

我们使用命令 docker build，从零开始来创建一个新的镜像。为此，我们需要创建一个 Dockerfile 文件，其中包含一组指令来告诉 Docker 如何构建我们的镜像。

设置镜像标签

我们可以使用 docker tag 命令，为镜像添加一个新的标签。

$ docker tag 5a32b6d303dd homerlim/ubuntu:v2

docker tag 镜像ID，这里是 5a32b6d303dd，用户名称、镜像源名(repository name)和新的标签名(tag)。
使用 docker images 命令可以看到，ID为 5a32b6d303dd 的镜像多一个标签。

$ docker images
REPOSITORY             TAG                 IMAGE ID            CREATED             SIZE

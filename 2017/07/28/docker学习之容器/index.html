docker学习之容器

发表于 2017-07-28 | 分类于 docker | 阅读次数:

容器的一些操作

创建容器

新建容器

可以使用 docker create 命令新建一个容器，例如：

root@default:~# sudo docker create -it ubuntu:latest
20fa1baa3f502c16aab764054fc7d177f8c7cc01e896b826b1ff7d70d6d1ae40

使用 docker create 命令新建的容器处于停止状态，可以使用 docker start 命令来启动它。

root@default:~# docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED             STATUS              PORTS               NAMES
20fa1baa3f50        ubuntu:latest       "/bin/bash"         24 seconds ago      Created                                 silly_wilson

root@default:~# docker start 20fa1baa3f50
20fa1baa3f50

root@default:~# docker ps -a
CONTAINER ID        IMAGE               COMMAND             CREATED              STATUS              PORTS               NAMES
20fa1baa3f50        ubuntu:latest       "/bin/bash"         About a minute ago   Up 51 seconds                           silly_wilson

新建容器并启动容器

启动容器有两种方式，一种是基于镜像新建一个容器并启动，另一个是将在终止状态的容器重新启动。所需要的命令为 docker run，等价于先执行 docker create 命令，再执行 docker start 命令。

root@default:~# sudo docker run ubuntu /bin/echo 'hello world'
hello world

这跟在本地直接执行/bin/echo ‘hello world’几乎感觉不出任何区别。命令执行完，容器自动终止。

下面的命令则启动一个 bash 终端，允许用户进行交互：

root@default:~# sudo docker run -t -i ubuntu /bin/bash
root@2ae1ab3d0c65:/#

其中，-t 选项让 docker 分配一个伪终端（pseudo-tty）并绑定到容器的标准输入上，-i 则让容器的标准输入保持打开。

在交互模式下，用户可以通过所创建的终端来输入命令，例如：

root@2ae1ab3d0c65:/# pwd
/

root@2ae1ab3d0c65:/# ls
bin  boot  dev  etc  home  lib  lib64  media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var

root@2ae1ab3d0c65:/# ps
    PID TTY          TIME CMD
    1 pts/0    00:00:00 bash
      10 pts/0    00:00:00 ps

在容器内用 ps 命令查看进程，可以看到，只运行了 bash 应用，并没收有运行其他不需要的进程。

用户可以按 Ctrl+d 或输入 exit 命令来退出容器：

root@2ae1ab3d0c65:/# exit
root@default:~#

对于所创建的 bash 容器，当使用 exit 命令退出之后，该容器就自动处于终止状态。

守护态运行

更多时候，需要让 docker 容器在后台以守护态形式运行。用户可以通过添加-d 参数来实现。

root@default:~# sudo docker run -d ubuntu /bin/sh -c "while true; do echo 'hello world'; sleep 5; done"
d7a6add38b0b250b3eb5f73460123d7a30fe5c8b69fcb2022892e2a06e062673

容器启动后返回一个唯一的 ID，也可以通过 docker ps 命令来查看容器信息：

root@default:~# docker ps
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS               NAMES
d7a6add38b0b        ubuntu              "/bin/sh -c 'while..."   47 seconds ago      Up 46 seconds                           elated_mcclintock

要获取容器的输出信息，可以通过 docker logs 命令：

root@default:~# docker logs d7a6add38b0b
hello world
hello world
hello world
root@default:~#

进入容器

在使用-d 参数时，容器启动后会进入后台，用户无法看到容器中的信息。某些时候如果需要进入容器进行操作，有多种方法，包括使用 docker attach 命令、docker exec 命令，以及 nsenter 工具等。

attach命令

docker attach 是 docker 自带的命令。以前面的命令为例，运行一个容器，隔5秒输出一次 hello world：

root@default:~# sudo docker run -d ubuntu /bin/sh -c "while true; do echo 'hello world'; sleep 5; done"
ac1faf393f35361736ed292b93fcd83935566a3517d2e14a54d30750587a5313

使用 docker attach 命令：

root@default:~# docker attach ac1faf393f35
hello world
hello world
hello world
hello world

但是使用 attach 命令有时并不方便。当多个容器同时 attach 到同一个容器时，所有容器都会同步显示。当某个容器因命令阻塞时，其他容器也无法执行操作了。

exec命令

使用 exec 命令，可以直接在容器内运行命令。例如，创建后台容器，并进入创建的容器中，并启动一个 bash:

docker@default:~$ sudo docker run -d ubuntu /bin/sh -c "while true; do echo 'hello world'; sleep 5; done"
a43a0a9085b1573b63ccf5964ced371a57fcbaa842a6b81a8e2618360ab56a44

docker@default:~$ sudo docker exec -ti a43a0a9085b1 /bin/bash
root@a43a0a9085b1:/#

可以在进入容器后使用命令，而不会像 attach 一样，会一直输出 hello world 了，例如：

root@a43a0a9085b1:/# ps -ef
UID        PID  PPID  C STIME TTY          TIME CMD
root         1     0  0 09:33 ?        00:00:00 /bin/sh -c while true; do echo 'hello world'; sleep 5; done
root        35     0  0 09:35 pts/0    00:00:00 /bin/bash
root       108     1  0 09:40 ?        00:00:00 sleep 5
root       109    35  0 09:40 pts/0    00:00:00 ps -ef

nsenter工具

使用 nsenter 连接到容器，然后与 exec 类似使用命令在容器内操作。

导入和导出容器

导出容器

导出容器，是指导出一个已经创建的容器到一个文件，不管此时这个容器是否处于运行状态，可以使用 docker export 命令，该命令格式为：docker export container。

使用 docker ps -a 查看所有容器，找到容器ID，使用命令导出容器，示例：

docker export a43a0a9085b1 > test_for_run.tar
docker export ac1faf393f35 > test_for_stop.tar

可将这些文件传输到其他机器上，在其他机器上通过导入命令实现容器的迁移。

导入容器

导出的文件又可以使用 docker import 命令导入，成为镜像，例如：

cat test_for_run.tar | sudo docker import - test/ubuntu:v1.0

导入后使用 docker images 查看镜像，应该可以看到导致的新镜像。

使用 docker load 命令可以导入镜像存储文件到本地的镜像库，也可以使用 docker import 命令来导入一个容器快照（例如本例中的 test_for_run.tar）到本地镜像库。这两者的区别在于容器快照文件将丢弃所有的历史记录和元数据信息（即仅保存容器当时的快照状态），而镜像存储文件将保存完整记录，体积也要大。此外，从容器快照文件导入时可以重新指定标签等元数据信息。

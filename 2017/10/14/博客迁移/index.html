博客迁移，从 github page 到 gitee page

发表于 2017-10-14 | 分类于 hexo | 阅读次数:

今天试了一下，把 github page 迁移到 gitee page。

修改 _config.yml

_config.yml 中相关配置项：

url: http://yoursite.com
root: /

在 github page，静态博客内容是放在根目录下的，所以不需要修改这个配置，用默认的就好，而在 gitee page，则是放在与仓库名同名的子目录下，根据配置文件里的注释：

## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'

需要做相应修改，改为（我为博客创建的仓库名为 mymemory）：

url: http://homerlim.gitee.io/mymemory
root: /mymemory/

另一个需要修改的，是提交的远程仓库地址，改为：

deploy:
    type: git
    repository: git@gitee.com:HOMERLIM/mymemory.git

本地图片插入方式调整

原来插入本地图片的方式是：在 source 目录下新建一个 img 文件夹，如果文章需要插入图片，则在img文件夹下新建一个与文章名字相同的文件夹，放入文章相关的图片，然后在文章中用以下方式引用图片：

![图片描述](/img/文章名/图片名.png)

这种方式生成的页面，图片路径在博客根目录下的 img 目录中，即生成静态博客内容时，source 下的 img 目录完全复制到博客根目录下，这在 github page 上是没问题的，因为 github page 的内容就是放在根目录下，图片按生成的路径可以正常访问，而 gitee page 访问不到（因为博客内容放在子目录中），图片不能在文章中正常显示出来。

现在要改成另一种方式，首先修改 _config.yml 的配置：

post_asset_folder: true        # 原来是false的，现改为true

这项配置修改后的结果是，每当用 hexo new post 生成新文章时，会自动创建与文章相同名称的文件夹，当文章中需要插入图片时，只需要图片放在该文件夹下就可以了，而且生成博客时，会自动将图片与文章的 .index 文件放在一起（不再是放在根目录下的 img 文件夹内），这种方式下，图片的引用方式是：

![图片描述](图片名字.png)

这样，图片就能正常地显示在文章中了。

但是，上面的做法，图片并不会显示在首页，如果有需要把图片显示在首页同时显示在文章中的需求，可以使用下面的引用方式：

{% asset_img 图片名.png 图片描述 %}

这不是正规的markdown语法，但是hexo支持该标签插件语法，这样生成的博客，图片即可以显示在首页，也可以正常显示在文章内。

阅读数量与评论配置

原来的博客中有配置阅读数量与评论，现在迁移后没有了，原因是 leancloud 配置了安全域名，不在安全域名列表中的域名，发起的请求会被忽略，所以只需要把新的博客地址加到安全列表就可以了，在leancloud安全中心中添加新的 web 安全域名。

最后，测试一下，生成的博客在 github page 和 gitee page 都可以正常显示，以后就可以写完生成，再提交到 gitee page，再同步到 github page 上了，或者干脆只在 gitee page 上写。

PS : 迁移到 gitee page 的原因，一个是 github 的访问速度原因，一个是 gitee page 的仓库可以不开放，而 github page 必须是开放的（指是的免费的仓库）。

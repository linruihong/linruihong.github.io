git的使用之ssh key

发表于 2017-10-10 | 分类于 git | 阅读次数:

使用远程仓库（如 github）时，使用 git push 命令推送新的修改到远程仓库时，一般都要求输入账号，密码，比较麻烦。省事的方法就是用 SSH key。

在本地生成 SSH key，然后把 SSH key 添加到远程仓库的账号中，相当于把本地机器加入到远程仓库的“白名单”中，之后再有 git push 操作时，就不再需要输入账号密码了。

具体做法是：

设置 git 使用的用户名与邮箱

git config --global user.email xxxxx@qq.com
git config --global user.name xxxxx

对应的删除已设置的用户名与邮箱命令，在修改用户名或者邮箱时使用

git config --global --unset user.email
git config --global --unset user.name

生成 SSH key

ssh-keygen -t rsa -C "xxxxx@qq.com"

密钥类型可以用 -t 选项指定。如果没有指定则默认生成用于 SSH-2 的 RSA 密钥。这里使用的是 rsa。同时在密钥中有一个注释字段，用 -C 来指定所指定的注释，可以方便用户标识这个密钥，指出密钥的用途或其他有用的信息。所以在这里输入自己的邮箱或者其他都行。

输入完毕后程序同时要求输入一个密语字符串(passphrase)，空表示没有密语。接着会让输入2次口令(password)，空表示没有口令。3次回车即可完成当前步骤，此时[c:/Users/Administrator/.ssh]目录下已经生成好了（生成目录不同用户不同机器不同系统版本下可能略有不同，以生成 SSH key 后，界面提示的生成目录为准）。.ssh 目录下有两个密钥文件 id_rsa 和 id_rsa.pub。

登录 github，把打开 Settings -> SSH and GPG keys，点 New SSH key，用记事本把 id_rsa.pub 打开（注意不要修改里面的内容），把里面的内容复制到 New SSH key 弹出窗口的 Key 输入框中，点 Add SSH key 保存，完成创建。

至此 SSH key 生成及添加完成。

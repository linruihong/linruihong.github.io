Visual Studio 2010

启动VS2010报错：The type initializer for '<Module>' threw an exception.

原因：之前安装过VS2010，可正常使用，之后删除VS2010打算装VS2012，可能删错东西或者删不干净，反正VS2012安装成功，但启动时也是报错：The type initializer for '<Module>' threw an exception.，最后删除VS2012，重新安装回VS2010，启动还是报这个错误：The type initializer for '<Module>' threw an exception.。

看到这个解决办法，只是尝试着把控制面板=>程序和功能中的所有Microsoft .NET Framework 4开头的程序删除掉，然后再以修复的方式重装VS2010，然后再启动就成功了。

神奇。。。

Mysql

因为要测试某些功能，而将服务器时间往后调，导致服务器登录不了，原来在线的也会掉线，查看发现CPU占用过高，是MYSQL占用了（接近100%）。

原因分析：服务器往后调整，使得服务器中原来是分布在不同时刻的任务，集中在同一个时间点执行，而这些任务中大部分会触发数据库读写，从而导致数据库服务占用了大部分的CPU资源。

具体是哪些操作导致的数据库频繁读写还不清楚，网上有介绍方法，说可以开启mysql慢查询日志功能来查看是哪些表操作占用时间较长。记一下，有空再试试。

批处理

最近整理电脑桌面，把工作的东西按目录整理了一下，能用批处理的尽量用批处理做了，例如更新版本、提交修改、导表等。现在开发工具是用PyCharm，每次打开一个工程，总是将工程目录拖放到PyCharm图标上，或者在File菜单中选择Open来打开，感觉麻烦，想用脚本来打开。

最开始的bat指令是这样：

"D:\MYAPP\JetBrains\PyCharm Community Edition 2017.1.2\bin\pycharm64.exe" ./game
  • .bat文件与工程目录放在同一级目录中
  • pycharm64.exe的目录用双引号括起来，是因为PyCharm工作目录中有空格，不用双引号括起来，会被误认为空格前是程序位置，空格后是命令参数
  • 使用pycharm64.exe而不是pycharm.exe是因为系统是64位的，用pycharm.exe会报错，无法打开工程

测试发现可以正确打开工程，但是总是等到打开的工程被关闭，.bat文件打开的CMD窗口才会跟着关闭，于是在网上找可以在启动程序后便关闭CMD窗口的方式，这是传送门，指令是这样的：

start "" "D:\MYAPP\JetBrains\PyCharm Community Edition 2017.1.2\bin\pycharm64.exe" ./game

这样就解决了问题了，666

Python 代码格式化工具 YAPF

安装：

pip install yapf

命令行下使用（ubuntu）：

yapf -i /pyfiles/helpy1.py

格式化前：

# -*- coding: utf-8 -*-

import sys
print(sys.getdefaultencoding())
if True:
        print("hello world")
        print (12345678)

格式化后：

# -*- coding: utf-8 -*-

import sys
print(sys.getdefaultencoding())
if True:
    print("hello world")
    print(12345678)

tab 会用空格替换，多余的空格会被删除等。。。

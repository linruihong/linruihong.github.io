hexo+github pages搭建、使用问题

发表于 2016-08-01 | 分类于 hexo | 阅读次数:

趁着有点空闲时间，把搭github pages以及使用过程遇到的问题记一记。（持续更新）

选择Hexo

Hexo并不是一开始就选择的。原本是在网上搜GO语言的开源项目，偶然看到一个静态博客生成器，于是想搞一搞，玩一玩，结果按照教程搭了一晚上也没搭出来（只能说那个教程不适合小白），弃了。然后第二天直接在网上搜静态博客生成器，结果出来一堆“github+Hexo轻松搭建个人博客”之类的文章，这才用了Hexo。搭建过程也确实够简单，够傻瓜，教程详细多了，这样很快就看到了第一篇“Hello world”。

文章创建日期问题

搞明白了搭建、换主题、发布，写新文章时，发现了新问题：每次修改文章，并使用 hexo g 命令生成页面，然后用 hexo s 在本地预览，文章的创建日期都会改变（变成文章的修改时间），解决办法是，在每篇文章的开头插入如下“固定开头”（包含标题，日期时间，标签可有可无）：

---
title: 搭建过程的问题
date: 2015-01-02
tags: 搭建
---

换主题

初始化 Hexo 时，自动下载的主题是官方的 Landscape，可以更换成其它主题。先在网上找到合适的主题，下载主题源码放到博客目录（ hexo init 初始化的那个目录）下的 source 目录，再把博客目录下 _config.yml 文件中 theme 属性后的 Landscape 替换成新主题的目录名字，再重新生成页面就可以了。

Hexo主题： https://github.com/hexojs/hexo/wiki/Themes
详细教程： http://theme-next.iissnan.com/getting-started.html#activate-next-theme

页面中文显示乱码

首先查看 _config.yml 里的 language 项是不是设置中文，如果不是则先设置对应的中文类型（例如： language: zh-Hans ），然后再重新生成页面。另外，在修改 _config.yml 文件后，要将文件另存为UTF-8格式，覆盖原来的文件，否则可能显示的中文依然是乱码。

部署 github pages

修改配置文件 _config.yml ，其中 repository 是远程仓库的地址，最好是 SSH 链接（如下例所示就是 SSH 链接），否则应该是会每次提交都要求输用户名和密码的。

deploy:
    type: git
    repository: git@github.com:lrhzone/lrhzone.github.io.git

注意上面的type: git不要写成type: github（Hexo 3.0以上版本）

使用下面的命令就可以将项目部署到github

hexo d

可能出现错误提示

ERROR Deployer not found: git

这是因为Hexo 3.0后，deploy git需要独立安装。
解决办法是，输入以下命令安装deploy git，然后就可以正常使用hexo d:

npm install hexo-deployer-git –save

参考：
http://blog.163.com/gis_warrior/blog/static/19361717320153100184696/
http://blog.csdn.net/wx_962464/article/details/44786929

标签/分类数量统计不准确

因为Hexo有缓存的功能，因此有时候会发现在【标签】和【分类】页面中的数量统计并不准确。出现这个问题时，可以按照以下步骤重新生成站点的内容：

  1. 删除站点目录下的 db.json 文件
  2. 在站点目录下执行命令 hexo clean
  3. 在站点目录下执行命令，重新生成页面 hexo generate

当执行完以上步骤后，可以在本地启动服务器来验证下是否已经解决问题。

实际上第1步是可以省去的，因为执行第2步后，窗口显示如下信息：

INFO  Deleted database.
INFO  Deleted public folder.

说明第2步实际包括了第1步的操作效果。

另外，貌似hexo在生成【标签】和【分类】页面中的数量统计时，会算上草稿中的定义的标签和分类，这个也会影响【标签】和【分类】页面中的数量统计的准确性，可以在未发布草稿前，可以先不要加上标签和分类。

插入图片

想在文章中插入图片，网上有人推荐七牛云存储，把图片存到云盘，然后把自动生成的图片链接使用到文章中。本来打算这样做的，但是注册要手机+实名认证，不想弄，于是测试下QQ相册的图片链接，发现可以，赞~

PS:现在也放弃了把图片放QQ空间中的方案了，毕竟不能保证那些空间能不能长久使用，说不定哪天停止服务呢

hexo 安装在哪里

安装完nodejs，新建目录 myblog，在blog下输入命令安装hexo

npm install -g hexo

结果不是装在 myblog 下，而是跑到C:\Users\Administrator\AppData\Roaming\npm\node_modules\hexo下面去了，不知道怎么回事，好在不影响使用。

查了下环境变量PATH中有这个目录C:\Users\Administrator\AppData\Roaming\npm，难道是因为这样，所以有后所有的新加模块都会到这里来？

在网上看到（注释）：

npm install hexo -g  #-g表示全局安装, npm默认为当前项目安装

估计是这个原因，所以没装在当前目录了。

如果选择-g全局安装，那如果删除了node，hexo应该也会删除？

Markdown编辑器

发表于 2016-07-28 | 分类于 markdown | 阅读次数:

工欲善其事，必先利其器，好的工具能事半功倍。既然要用Markdown写博客，那就要找个好的编辑器了。

我选择Markdown编辑器，经历了从MarkdownPad，到简书，到Cmd Markdown，最后再回到MarkdownPad，过程太折腾了。

1. MarkdownPad

Windows平台的编辑器，是我首先选的也是最终选的编辑器，离线编辑器，带预览功能，可以边写边看显示效果，没试过其他的离线编辑器，因为感觉这个已经足够好了。

Windows 10下安装有问题，预览界面会提示This view has crashed，刚开始以为是免费版的问题，差点就放弃了这个编辑器，隔了几天偶然在网上看到别人也有这个问题，并且也解决了，按照给的方法，现在可以正常使用了，赞！

解决方法： http://www.07net01.com/2015/09/922578.html

经过选择Markdown编辑器这件事，使我明白，没事别升Windows 10

2. 简书

这是一个网络版的编辑器，提供在线编辑，及一键导出功能，基本就是个现成的博客系统。默认不是按Markdown格式编写文章的，如果需要，则要设置成Markdown格式才行，不然导出来的文章可能不是Markdown格式咯。

注册不用绑定手机号，但如果想发布文章或者导出文章内容，则需要绑定手机号。

最后没选择用这个编辑器，主要是嫌备份文章麻烦，要先导出，再提交到git仓库，没有本地编写，一个命令提交来得省事。（另一个原因是刚开始用它时，文件导出功能有问题，导出的文件一堆乱码，所以就不用了，后来隔几天再导，才能正常导出。）

虽然没用来写博客，但是下了个App在手机里，当看书软件用还是可以的。

3. Cmd Markdown

这是另一个网络版编辑器，纯Markdown格式编写，实时预览。文件导出貌似只能单个导出，不过不影响使用，而且注册不用绑手机啊！正式开始写博客用的就是它，如果不是后来发现MarkdownPad的问题可以解决，应该会一直用下去。

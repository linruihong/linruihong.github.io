git的使用之git config

发表于 2016-07-27 | 分类于 git | 阅读次数:

好记性不如烂笔头，何况记性又不好。最近为了玩github pages，使用git的频率有点高，可是就是记不住那些命令，用的时候经常要在网上搜一遍。所以就想将一些常用到的挑出来，归类写在这里，以后找起来也方便些，从git config开始吧！

设置用户名、邮箱

参考资料:http://blog.csdn.net/sky1203850702/article/details/40985505

安装完git，使用前，要先设置用户名和邮箱，否则提交代码提示:

*** Please tell me who you are.
Run
git config --global user.email "you@example.com"
git config --global user.name "Your Name"
to set your account's default identity.
Omit --global to set the identity only in this repository.
fatal: empty ident name (for <(null)>) not allowed

全局设置

顾名思义，就是设置的用户名全局有效，当提交某个目录时，如果该目录没有设置用户名时，就会默认使用全局用户名进行提交，设置全局用户名的命令如下:

git config --global user.name yourname
git config --global user.email youremail

“–global”（global前面是两个“-”，应该是Hexo主题的原因导致有一个显示不出来→_→）是表示这个设置是全局有效的，而user.name则是在使用git提交代码到远程仓库时，显示的用户名。

本地设置

在同一台电脑上，有多个远程仓库的代码，如果想在提交不同仓库代码时显示不同用户名，那就要对不同仓库设置不同的user.name。具体做法是，进入仓库代码所在目录，执行如下命令：

git config --local user.name yourname
git config --local user.email youremail

这个设置只在当前目录下有效，在别的目录不生效。而加了–global选项的命令，在任何目录下设置都视为全局设置。

  • 如果一如电脑上设置了global用户名，在某个目录下还设置了local用户名，则当提交该目录时，使用的是local用户名。
  • 如果一如电脑上设置了global用户名，而某个目录下没有设置local用户名，则当提交该目录时，使用的是global用户名。

查看配置

查看当前目录配置信息，bash进入当前目录，输入以下命令:

git config --list
git config --local --list #带local参数和不带参数显示不同

查看全局配置信息，bash直接输入以下命令:

git config --global --list

更改、删除用户名

如果当前使用的用户名不想要了，或者想换掉，可以使用下面命令:

git config --local --unset user.name #删掉当前目录的user.name
git config --global --unset user.name #删除全局设置的user.name

同理，删除user.email也是使用类型命令:

git config --local --unset user.email
git config --global --unset user.email
# git config

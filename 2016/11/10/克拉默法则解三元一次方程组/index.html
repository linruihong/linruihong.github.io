克拉默法则解三元一次方程组

发表于 2016-11-10 | 分类于 algorithm | 阅读次数:

数学这东西对搞机的人来说，真的是太重要了。当年学的线性代数，都还给老师了。

⊙﹏⊙‖∣

“复习”线性代数中的克拉默法则，还是上回做系统组队规则的事，当时要解三元一次方程组，又不知道怎么实现，于是到网上找算法，无意间看到了别人写的解方程计算器，里面用到了克拉默法则，果断“复习”了一遍。^_^

已知：
    单位分三种类型A，B，C，三种类型的数量为a，b，c
    队伍成员组成方式有3种，ABCAA，ABCAB，ABAAA
求：
    怎么分配3种组成方式的比例，使得尽可能让队伍成员组成落在这三种方式内，
    而不会出现AAAAA，BBBBB，CCCCC类似这种奇葩队伍。

这里只能说使队伍尽量组成3种组成方式中的其中一种，因为最终组成什么样的队伍，还要受制于a，b，c这三个数量。

假设每个队伍都能组满，队伍数量总数N，其中3种组成方式的比例分别是x、y、z，那么可以建立如下方程：

x+y+z = 100% = 1
(a+b+c)/5 = N
x*N*3 + y*N*2 + z*N*4 = a
x*N*1 + y*N*2 + z*N*1 = b
x*N*1 + y*N*1 + z*N*0 = c

简化一下：

x+y+z = 100% = 1
x*3 + y*2 + z*4 = a/N = 5a/(a+b+c)
x*1 + y*2 + z*1 = b/N = 5b/(a+b+c)
x*1 + y*1 + z*0 = c/N = 5c/(a+b+c)

取后三个方程，组成方程组，再利用克拉默法则，就可以解出x，y，z了。

由于a、b、c的不确定性，所以不同的a、b、c有可能导致x、y、z中存在负值，而队伍组成方式的比例必须是大于或者等于0，所以在遇到负数的情况下，直接就舍弃掉，即不会出现该组成方式的队伍，这是根据实际情况做的特殊处理。

克拉默法则解方程组的代码如下：

# -*- coding: gbk -*-

'''
本模块解三元一次方程组
'''

def CalcThirdOrderDeterminant(tomd):
    #本函数计算三阶行列式
    #tomd(third order matrix data)是三阶矩阵数据：
    #[[a11, a12, a13], 
    # [a21, a22, a23], 
    # [a31, a32, a33]]
    a11a22a33 = tomd[0][0] * tomd[1][1] * tomd[2][2]
    a12a23a31 = tomd[0][1] * tomd[1][2] * tomd[2][0]
    a13a21a32 = tomd[0][2] * tomd[1][0] * tomd[2][1]
    a11a23a32 = tomd[0][0] * tomd[1][2] * tomd[2][1]
    a12a21a33 = tomd[0][1] * tomd[1][0] * tomd[2][2]
    a13a22a31 = tomd[0][2] * tomd[1][1] * tomd[2][0]
    D = a11a22a33 + a12a23a31 + a13a21a32 - a11a23a32 - a12a21a33 - a13a22a31
    return D

def SolveEquation(colst):
    #本函数克拉默法则解三元一次方程组，返回4个行列式D,D1,D2,D3的值
    #最终的三个解分别是：D1/D、D2/D、D3/D
    #a11 * X + a12 * Y + a13 * Z = b1
    #a21 * X + a22 * Y + a23 * Z = b2
    #a31 * X + a32 * Y + a33 * Z = b3
    #colst：方程组系数[[a11,a12,a13,b1],[a21,a22,a23,b2],[a31,a32,a33,b3]]
    if len(colst) != 3:
        return

    for co in colst:
        if len(co) != 4:
            return

    tomd = []
    for lst in colst:
        tomd.append([lst[0], lst[1], lst[2]])
    D = CalcThirdOrderDeterminant(tomd)

    if D == 0: #无解
        print "无解"
        return

    tomd = []
    for lst in colst:
        tomd.append([lst[3], lst[1], lst[2]])
    D1 = CalcThirdOrderDeterminant(tomd)

    tomd = []
    for lst in colst:
        tomd.append([lst[0], lst[3], lst[2]])
    D2 = CalcThirdOrderDeterminant(tomd)

    tomd = []
    for lst in colst:
        tomd.append([lst[0], lst[1], lst[3]])
    D3 = CalcThirdOrderDeterminant(tomd)

    return (D,D1,D2,D3)

if __name__ == '__main__':
    print SolveEquation([[3,2,4,60.2],[1,2,1,23.0],[1,1,0,16.8]])
